## KRITISCHE Constraints fürs nächste Mal (Deployment)
- **`run.ps1` MUSS reines ASCII + UTF-8-BOM sein** (Ziel: Windows PowerShell **5.1**).
  - Em-Dashes/Umlaute **ohne** BOM → PS 5.1 liest cp1252 → „unexpected )/}".
  - **Nicht blind mit dem Edit-Tool bearbeiten** (kann das BOM strippen)! Stattdessen:
    `[System.IO.File]::ReadAllText` → ASCII-Replace → `WriteAllText(path, text, (New-Object System.Text.UTF8Encoding($true)))`. Danach Bytes (EF BB BF) + Parser prüfen.
- **stderr unter `$ErrorActionPreference='Stop'`**: native stderr (z. B. Flask-Dev-Warnung) bricht in PS 5.1 ab → in `Start-App` lokal `Continue` (try/finally).
- **run.ps1 churn-arm:** Version (Laufzeit-Lookup) + App-Name (aus config.py) + neue Wheels berühren run.ps1 **nicht** → seltene Re-Signaturen. Nur App-Logik/Python-Version-Gate ändern es.
- **Release bauen:** `python tools/build_release.py` → `build/<Name>-<ver>.zip`; `gh release create vX.Y.Z <zip> --title … --notes …`. `run.ps1` ist **unsigniert** im Paket (Nutzer signiert nach dem Entpacken).
