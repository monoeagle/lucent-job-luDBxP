# Session Handoff — 2026-06-25 22:23

## Was wir gemacht haben
Neues Projekt **Lucent DB Explorer** (lucent-job-luDBxP) von Null aufgebaut:
ein Flask-Web-Tool, das die Frage „Wie komme ich von Tabelle A zu B?"
beantwortet — Schema reflektieren → FK-Graph → kürzeste Join-Pfade + SQL,
visualisiert und interaktiv. Brainstorming → Spec → Plan → 10 TDD-Tasks
(Subagent-getrieben) für v1, danach mehrere Feature-Erweiterungen.

      (deterministisch) → read-only, parametrisiertes SQL. Flask-API + UI.
- [x] Filter-UI (WHERE über Zwischentabellen, UND-verknüpft)
- [x] Graph-Visualisierung (Cytoscape.js, lokal gebundelt; Pfad-Highlight)

- [x] Datenvorschau (`/api/data`, read-only SELECT..LIMIT, Objekt validiert)

## Entscheidungen dieser Session
- **Stack:** Flask-Web (nicht PyQt6-Desktop) — die Idee ist eine Browser-UI;
  kein fertiges `.pattern`, abgeleitet aus der `-mpp`-Flask-Linie.
- **Verifikation:** Playwright (System-python3) für die JS-UI, da pytest nur
  Backend testet. Screenshots als Beleg.
- **Read-only nuanciert:** Join-SQL nie ausführen; Datenvorschau führt
  kontrolliertes read-only SELECT aus (Objekt gegen Schema validiert).
- **Implied FKs** als Haupt-Mehrwert gegenüber SchemaSpy (CMDBs ohne FKs).
